01
Algorithme de Prim

// Explication & Fonctionnement

Objectif : Trouver l'Arbre Couvrant Minimum (ACM) d'un graphe connexe non orienté pondéré — connecter tous les sommets avec le poids total minimum, sans cycle.

Idée centrale : Prim construit l'arbre sommet par sommet. À chaque étape, il choisit l'arête de poids minimum reliant un sommet déjà dans l'arbre à un sommet extérieur.

  1. Choisir un sommet de départ arbitraire, le marquer comme "visité".
  2. Ajouter toutes les arêtes de ce sommet dans une file de priorité (min-heap).
  3. Extraire l'arête de poids minimum. Si le sommet destination est déjà visité → ignorer.
  4. Sinon, ajouter ce sommet à l'arbre, ajouter ses arêtes voisines dans la file.
  5. Répéter jusqu'à ce que tous les sommets soient dans l'arbre (n-1 arêtes).
O((V+E) log V) avec min-heap Non orienté · Pondéré Condition : connexe
Cas Normaux
Normal Graphe simple connexe (5 sommets)
▼
Arêtes : A,B,2 A,C,3 B,C,1 B,D,4 C,D,5 C,E,6 D,E,2
✓ Résultat attendu

ACM : B—C(1), A—B(2), D—E(2), B—D(4) → Poids total = 9. n-1 = 4 arêtes pour 5 sommets ✓

⚠ Ce qu'il faut vérifier

L'algorithme peut démarrer de n'importe quel sommet et doit toujours donner le même poids total (même si les arêtes peuvent varier en cas d'égalité).

✓ Résultat attendu

✓ Résultat attendu

⚠ Détection

Après l'exécution, si le nombre d'arêtes dans l'ACM < n-1, le graphe n'est pas connexe.

02
Algorithme de Kruskal

// Explication & Fonctionnement

Objectif : Même objectif que Prim — trouver l'ACM — mais approche radicalement différente.

Idée centrale : Kruskal construit l'arbre arête par arête. Il trie toutes les arêtes par poids croissant et les ajoute une par une, en rejetant celles qui créeraient un cycle. Utilise la structure Union-Find (DSU) pour détecter les cycles en O(α(n)).

  1. Trier toutes les arêtes par poids croissant.
  2. Initialiser n composantes connexes (chaque sommet est sa propre composante).
  3. Pour chaque arête (u,v) dans l'ordre : si u et v sont dans des composantes différentes → ajouter à l'ACM et fusionner (Union).
  4. Si u et v sont dans la même composante → ignorer (créerait un cycle).
  5. S'arrêter quand l'ACM contient n-1 arêtes.
O(E log E) — dominé par le tri Non orienté · Pondéré Meilleur sur graphes épars
Cas Normaux
Normal Graphe classique (même que Prim — vérification croisée)
▼
Arêtes (triées) : B,C,1 A,B,2 D,E,2 A,C,3 B,D,4 C,D,5 C,E,6
✓ Résultat attendu

ACM : {B—C(1), A—B(2), D—E(2), B—D(4)} → Poids = 9. Même résultat que Prim ✓

⚠ Test de cohérence

Comparer le résultat de Prim et Kruskal sur le même graphe : les poids totaux doivent être identiques.

Cas Délicats
Délicat Graphe circulaire C5 (rejeter l'arête la plus lourde du cycle)
▼
Arêtes : A,B,5 B,C,3 C,D,2 D,E,6 E,A,4
✓ Résultat attendu

ACM : {C—D(2), B—C(3), E—A(4), A—B(5)} → exclure D—E(6). Poids = 14.

Délicat Graphe complet K5 (beaucoup d'arêtes inutiles)
▼
Arêtes : A,B,10 A,C,8 A,D,6 A,E,4 B,C,7 B,D,5 B,E,3 C,D,9 C,E,2 D,E,1
✓ Résultat attendu

ACM : {D—E(1), C—E(2), B—E(3), A—E(4)} → Poids = 10. 6 arêtes rejetées.

Délicat Union-Find : étoile centrale (compression de chemin)
▼
Arêtes : B,C,1 C,D,1 D,E,1 E,F,1 A,B,1 A,C,2 A,D,3 A,E,4 A,F,5
✓ Résultat attendu

ACM = {B—C(1), C—D(1), D—E(1), E—F(1), A—B(1)} → Poids = 5. Tester que find() avec compression de chemin ne cause pas d'erreurs.

Délicat Toutes les arêtes de poids égal (ACM non unique)
▼

✓ Résultat attendu

✓ Résultat attendu

03
Chemins & Circuits Eulériens

// Explication & Fonctionnement

Définitions :
— Chemin eulérien : passe par chaque arête exactement une fois (sans nécessairement revenir au départ).
— Circuit eulérien : chemin eulérien qui revient au sommet de départ.

Conditions (graphe non orienté) :

  1. Circuit eulérien : graphe connexe + tous les sommets ont un degré pair.
  2. Chemin eulérien : graphe connexe + exactement 2 sommets ont un degré impair (départ et arrivée).

Pour graphe orienté : Circuit : in-deg = out-deg pour tout sommet. Chemin : un sommet a out-deg − in-deg = 1 (départ), un a in-deg − out-deg = 1 (arrivée).

Algorithme de Hierholzer : partir d'un sommet, suivre les arêtes jusqu'à revenir au départ, puis insérer les sous-circuits non traités. Complexité O(V + E).

O(V + E) Orienté ou Non orienté
Cas Normaux
Normal Circuit eulérien classique (tous degrés pairs)
▼
Arêtes : A,B,1 B,C,1 C,A,1 A,D,1 D,B,1 B,A,1 // Degrés : A=4, B=4, C=2, D=2 → tous pairs ✓
✓ Résultat attendu

Circuit eulérien valide, ex : A→B→C→A→D→B→A. Toutes les 6 arêtes parcourues exactement une fois, retour au départ ✓

Normal Chemin eulérien (exactement 2 sommets de degré impair)
▼
Arêtes : A,B,1 A,C,1 B,C,1 B,D,1 C,E,1 D,E,1 // Degrés : A=2, B=3, C=3, D=2, E=2 // B et C ont degré impair → chemin de B à C
✓ Résultat attendu

Chemin eulérien de B à C (ou C à B). Ex : B→A→C→B→D→E→C. 6 arêtes parcourues, début ≠ fin.

Normal Circuit eulérien orienté (in-deg = out-deg)
▼
Arcs (orientés) : A,B,1 B,C,1 C,A,1 // in-deg = out-deg = 1 pour tous → circuit eulérien orienté
✓ Résultat attendu

Circuit eulérien orienté : A→B→C→A. Les conditions pour graphe orienté diffèrent du non orienté.

Cas Sans Solution
Sans Solution Plus de 2 sommets de degré impair (Königsberg)
▼
Arêtes : A,B,1 A,B,1 A,C,1 A,C,1 A,D,1 B,D,1 C,D,1 // Degrés : A=5, B=3, C=3, D=3 → 4 sommets impairs
✓ Résultat attendu

Ni chemin ni circuit eulérien. Message : "Aucun chemin/circuit eulérien. 4 sommets ont un degré impair (≠ 0 ou 2)."

Sans Solution Graphe non connexe avec degrés pairs
▼
Arêtes : A,B,1 B,C,1 C,A,1 D,E,1 E,F,1 F,D,1 // Tous degrés pairs MAIS graphe non connexe
✓ Résultat attendu

Pas de circuit eulérien malgré les degrés pairs. La connexité doit être vérifiée AVANT les conditions de degré.

⚠ Piège fréquent

Vérifier uniquement les degrés sans tester la connexité est une erreur classique — un programme naïf retournerait un faux positif.

Cas Extrêmes & Délicats
Extrême Graphe à 1 seule arête
▼
Arêtes : A,B,1 // Degrés : A=1, B=1 → 2 sommets impairs
✓ Résultat attendu

Chemin eulérien : A→B. Pas de circuit. 1 arête parcourue.

Délicat Sommet isolé dans un graphe eulérien
▼
Arêtes : A,B,1 B,C,1 C,A,1 // D est déclaré mais isolé (degré 0)
✓ Résultat attendu

Selon la définition : le graphe restreint aux arêtes est connexe et eulérien. Si D est inclus dans le graphe global → non connexe. Définir le comportement attendu avec l'équipe.

⚠ Définition à clarifier

Certaines implémentations ignorent les sommets de degré 0 pour tester la connexité — d'autres non.

04
Coloration Welsh-Powell

// Explication & Fonctionnement

Objectif : Colorer les sommets avec le minimum de couleurs de sorte que deux sommets adjacents n'aient jamais la même couleur. Welsh-Powell est un algorithme glouton (heuristique), pas forcément optimal.

  1. Calculer le degré de chaque sommet.
  2. Trier les sommets par degré décroissant.
  3. Prendre la première couleur. Parcourir la liste triée : attribuer cette couleur à tous les sommets non adjacents à un sommet déjà coloré avec cette couleur.
  4. Passer à la couleur suivante. Répéter pour les sommets non encore colorés.
  5. S'arrêter quand tous les sommets sont colorés.

Note : Welsh-Powell ne garantit pas le nombre chromatique χ(G) optimal, mais donne une bonne approximation en O(V² + E).

O(V² + E) Non orienté Heuristique (non optimal garanti)
Cas Normaux
Normal Graphe biparti K(3,3) — 2 couleurs suffisent
▼
Arêtes : A,D,1 A,E,1 A,F,1 B,D,1 B,E,1 B,F,1 C,D,1 C,E,1 C,F,1
✓ Résultat attendu

χ(G) = 2. Welsh-Powell doit trouver 2 couleurs : {A,B,C} = Couleur1, {D,E,F} = Couleur2.

✓ Résultat attendu

χ(C6) = 2. Coloration : {1,3,5} = Rouge, {2,4,6} = Bleu.

Cas Extrêmes
Extrême Graphe complet K5 — 5 couleurs obligatoires
▼
Arêtes : A,B,1 A,C,1 A,D,1 A,E,1 B,C,1 B,D,1 B,E,1 C,D,1 C,E,1 D,E,1
✓ Résultat attendu

χ(K5) = 5. Chaque sommet a une couleur différente. Welsh-Powell est optimal ici (pas d'autre solution).

Extrême Graphe sans arête — 1 couleur
▼
Sommets : A, B, C, D, E Arêtes : aucune
✓ Résultat attendu

χ = 1. Une seule couleur pour tous les sommets (aucun conflit possible).

Cas Délicats
Délicat Cycle impair C5 — 3 couleurs minimum
▼
Arêtes : A,B,1 B,C,1 C,D,1 D,E,1 E,A,1 // Tous les degrés = 2 → ordre arbitraire
✓ Résultat attendu

χ(C5) = 3. Exemple : A=1, B=2, C=1, D=2, E=3.

⚠ Piège

Tous les degrés sont identiques → l'ordre de tri n'est pas déterministe. Le résultat peut varier selon l'implémentation, mais doit toujours utiliser 3 couleurs.

Délicat Welsh-Powell non optimal (cas connu — heuristique)
▼
Arêtes : 1,2,1 1,3,1 2,4,1 2,5,1 3,4,1 3,6,1 4,5,1 4,6,1 5,6,1
✓ Résultat attendu

χ réel = 3. Welsh-Powell peut retourner 3 ou 4 couleurs selon l'ordre de tri. Ce n'est pas un bug — documenter ce comportement heuristique dans les tests.

05
Ford-Fulkerson — Flot Maximum

// Explication & Fonctionnement

Objectif : Trouver le flot maximum pouvant être envoyé d'une source S à un puits T dans un réseau orienté avec capacités.

Idée centrale : Augmenter itérativement le flot via des chemins augmentants (S→T avec capacité résiduelle). Le graphe résiduel modélise les capacités restantes ET les flots inverses pour annuler de mauvais choix.

  1. Initialiser tous les flots à 0. Construire le graphe résiduel.
  2. Trouver un chemin augmentant de S à T dans le graphe résiduel (BFS = Edmonds-Karp, ou DFS = Ford-Fulkerson classique).
  3. Trouver le goulot d'étranglement (capacité minimale sur ce chemin).
  4. Augmenter le flot sur ce chemin, mettre à jour le graphe résiduel (capacité directe −δ, capacité inverse +δ).
  5. Répéter jusqu'à l'absence de chemin augmentant → flot maximum atteint.

Théorème Max-flow Min-cut : Flot maximum = Coupe minimale. Vérifier ce résultat pour valider l'implémentation.

O(E × |f*|) FF classique O(VE²) Edmonds-Karp (BFS) Orienté · Capacités
Cas Normaux
Normal Réseau simple 4 nœuds (2 chemins parallèles)
▼
Source : S Puits : T Arcs (capacités) : S,A,10 S,B,10 A,T,10 B,T,10 A,B,1
✓ Résultat attendu

Flot max = 20. S→A→T (flot 10) + S→B→T (flot 10). L'arc A→B(cap 1) est inutilisé.

Normal Réseau avec goulot d'étranglement central
▼
Arcs : S,A,15 S,B,4 A,C,12 B,C,10 C,T,7 A,T,5
✓ Résultat attendu

Flot max = 12. Vérification coupe min : {C→T + A→T} = 7+5 = 12 ✓

Cas Sans Solution / Flot = 0
Sans Solution Aucun chemin de S à T
▼
Arcs : S,A,10 B,T,10 // A et B ne sont pas connectés
✓ Résultat attendu

Flot max = 0. Message : "Aucun chemin de S à T. Flot = 0." Pas de crash.

Sans Solution Capacités toutes à 0
▼
Arcs : S,A,0 A,T,0
✓ Résultat attendu

Flot max = 0. Les arcs existent mais ne peuvent transporter aucun flot.

Cas Délicats
Délicat Arcs inverses cruciaux (graphe résiduel)
▼
Arcs : S,A,5 S,B,5 A,B,3 A,T,4 B,T,4 // Si DFS choisit S→A→B→T d'abord (flot 3), // l'arc résiduel B→A doit permettre d'annuler ce choix
✓ Résultat attendu

Flot max = 8. Chemins optimaux : S→A→T(4) + S→B→T(4).

⚠ Point critique

Sans capacités inverses correctes, l'algorithme peut rester bloqué à flot = 7. Vérifier que le graphe résiduel est mis à jour après chaque augmentation.

Délicat Flots inverses — annuler un mauvais choix initial
▼
Arcs : S,A,3 S,B,3 A,B,3 A,T,2 B,T,3 // Si DFS choisit S→A→B→T (flot 3), // il doit ensuite utiliser l'arc résiduel B→A
✓ Résultat attendu

Flot max = 5. Chemins optimaux : S→A→T(2) + S→B→T(3).

⚠ Piège majeur

Sans les arcs inverses dans le graphe résiduel, FF reste bloqué à une solution sous-optimale. C'est LE bug le plus fréquent dans les implémentations de Ford-Fulkerson.

Délicat Arcs antiparallèles (A→B et B→A simultanément)
▼
Arcs : S,A,10 A,T,10 A,B,5 B,A,5 B,T,5 S,B,5
✓ Résultat attendu

Flot max = 15. Les arcs antiparallèles peuvent confondre le graphe résiduel si mal implémenté.

⚠ Implémentation

Avec une matrice d'adjacence, A→B et B→A s'interfèrent. Il faut une liste d'adjacence avec indices d'arcs pour gérer correctement ce cas.

Délicat Chemins multiples (de nombreuses routes S→T)
▼

✓ Résultat attendu

Cas Extrêmes
