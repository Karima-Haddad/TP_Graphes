Contrat 01

Contrat global d’exécution

Ce contrat est la base commune pour toute l’équipe. Chaque exécution d’algorithme reçoit un nom d’algorithme, un graphe et des paramètres. La réponse renvoie ensuite le résultat logique, la visualisation finale et les étapes d’exécution.

Règle générale
Chaque algorithme doit suivre la même enveloppe JSON. Seuls params, result.summary et result.details changent selon l’algorithme.
  • algorithm : nom technique de l’algorithme
  • graph : structure du graphe d’entrée
  • params : paramètres propres à l’algorithme
  • result : réponse finale logique
  • visualization.result_graph : état final à dessiner
  • visualization.steps : étapes pour les boutons suivant / précédent / lecture / pause / stop
Requête standard
{
  "algorithm": "dijkstra",
  "graph": {
    "directed": false,
    "weighted": true,
    "nodes": [
      { "id": "A", "label": "A", "x": 120, "y": 80 },
      { "id": "B", "label": "B", "x": 290, "y": 130 },
      { "id": "C", "label": "C", "x": 110, "y": 210 },
      { "id": "D", "label": "D", "x": 320, "y": 240 },
      { "id": "E", "label": "E", "x": 430, "y": 190 }
    ],
    "edges": [
      { "id": "e1", "source": "A", "target": "B", "weight": 4 },
      { "id": "e2", "source": "A", "target": "C", "weight": 2 },
      { "id": "e3", "source": "B", "target": "D", "weight": 5 },
      { "id": "e4", "source": "C", "target": "D", "weight": 1 },
      { "id": "e5", "source": "D", "target": "E", "weight": 3 }
    ]
  },
  "params": {
    "source": "A",
    "target": "E"
  }
}
Contrat 02

Contrat standard de structure du graphe

Ce contrat garantit que tous les algorithmes reçoivent le même format d’entrée, et que le frontend peut garder les mêmes positions visuelles entre le graphe initial, les étapes et le graphe résultat.

Structure complète
{
  "directed": true,
  "weighted": true,
  "nodes": [
    {
      "id": "A",
      "label": "A",
      "x": 120,
      "y": 80,
      "data": {}
    }
  ],
  "edges": [
    {
      "id": "e1",
      "source": "A",
      "target": "B",
      "weight": 4,
      "label": "4",
      "data": {}
    }
  ]
}
Règles de validation
directed
true = orienté, false = non orienté
weighted
true = pondéré, false = non pondéré
node.id
obligatoire et unique
edge.id
obligatoire et unique
source / target
doivent référencer des nœuds existants
x / y
fortement recommandés pour garder la stabilité visuelle
weight
requis si le graphe est pondéré, sinon facultatif
Pour un graphe non orienté, le backend et le frontend doivent considérer que (A,B) et (B,A) représentent la même connexion logique.
Contrat 03

Contrat standard de succès

Ce format s’applique quand l’algorithme s’exécute correctement. Il sépare la logique du résultat final et la partie visuelle nécessaire à l’affichage et à la lecture pas à pas.

Réponse standard de succès
{
  "success": true,
  "algorithm": "dijkstra",
  "message": "Exécution réussie",
  "params": {
    "source": "A",
    "target": "E"
  },
  "result": {
    "summary": {},
    "details": {}
  },
  "visualization": {
    "result_graph": {
      "highlighted_nodes": [],
      "highlighted_edges": [],
      "node_colors": {},
      "edge_colors": {},
      "node_labels": {},
      "edge_labels": {}
    },
    "steps": []
  },
  "meta": {
    "execution_time_ms": 12,
    "step_count": 5,
    "warnings": []
  },
  "error": null
}
Contrat 04

Format standard des étapes

Les étapes sont utilisées uniquement pour l’interface d’exploration du processus. Elles permettent de piloter le graphe étape par étape avec les boutons précédent, suivant, lecture, pause et stop.

Objet étape
{
  "index": 0,
  "title": "Initialisation",
  "description": "Distance(A)=0, les autres à infini",
  "state": {
    "highlighted_nodes": ["A"],
    "highlighted_edges": [],
    "visited_nodes": [],
    "selected_nodes": [],
    "selected_edges": [],
    "node_labels": {
      "A": "0",
      "B": "∞"
    },
    "edge_labels": {},
    "node_colors": {
      "A": "#4f46e5"
    },
    "edge_colors": {},
    "extra": {}
  }
}
Signification des champs
  • highlighted_nodes : nœuds mis en évidence à l’étape courante
  • highlighted_edges : arêtes mises en évidence à l’étape courante
  • visited_nodes : nœuds déjà visités
  • selected_nodes : nœuds retenus dans le résultat courant
  • selected_edges : arêtes retenues dans le résultat courant
  • node_labels : distance, ordre, couleur, rang…
  • edge_labels : capacité, poids modifié, flot…
  • extra : informations spécifiques à l’algorithme
Le frontend lit visualization.steps[currentStep] et redessine le graphe avec son state.
Contrat 05

Contrat standard d’erreur

Ce format est renvoyé si le graphe est invalide, si les paramètres sont incorrects, ou si l’algorithme n’est pas applicable au graphe fourni.

Réponse standard d’erreur
{
  "success": false,
  "algorithm": "dijkstra",
  "message": "Le graphe contient un poids négatif, incompatible avec Dijkstra",
  "params": {
    "source": "A",
    "target": "E"
  },
  "result": null,
  "visualization": null,
  "meta": {
    "execution_time_ms": 0,
    "step_count": 0,
    "warnings": []
  },
  "error": {
    "code": "INVALID_GRAPH_FOR_ALGORITHM",
    "type": "validation_error",
    "field": "graph.edges[2].weight",
    "details": {
      "algorithm_requires": "positive_weights_only",
      "received_weight": -3,
      "edge_id": "e3"
    }
  }
}
Codes d’erreur recommandés
  • INVALID_GRAPH
  • INVALID_PARAMS
  • NODE_NOT_FOUND
  • EDGE_NOT_FOUND
  • INVALID_GRAPH_FOR_ALGORITHM
  • NEGATIVE_CYCLE_DETECTED
  • NO_EULERIAN_PATH
  • NO_SOLUTION
  • INTERNAL_ERROR
